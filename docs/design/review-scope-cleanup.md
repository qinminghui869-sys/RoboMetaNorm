# Review 范围精简设计

## 背景

字段规范化结果中出现了 `missing_collection_program` 和 `missing_conversion_script` 两条通用复核项。它们通过数据集目录中的脚本文件名推测采集与转换准备情况，但不参与相机或机器字段的语义、结构、单位及目标名称判断。

当前工具的职责是生成非破坏性的字段命名建议，而不是验证数据采集工程或转换工程是否随数据集交付。因此，这两项检查会产生与当前职责无关的人工复核，并带来额外目录遍历。

## 变更范围

- 删除采集程序存在性检查。
- 删除 LeRobot 转换脚本存在性检查。
- 删除仅被上述检查使用的脚本扩展名常量和文件名搜索函数。
- 删除测试夹具中为满足上述检查而创建的空脚本。
- 增加回归测试：字段和媒体证据完整时，不要求数据集包含采集或转换脚本，检查结果应为 `PASS`。

## 保留行为

以下内容不在本次变更范围内：

- RGB、action、observation 和视觉特征等字段命名必要证据检查。
- 相机媒体、命名冲突和 VLM 复核。
- 夹爪方向、量程和单位复核。
- 灵巧手字段的保留源名与人工复核行为。
- 输出结构、状态优先级和源数据只读保证。

因此，包含夹爪不确定项的数据集仍可保持 `REVIEW`；本次变更只移除与字段命名无关的两类复核。

## 验证

1. 先新增无脚本数据集的前置检查测试，并确认旧实现产生 `REVIEW`。
2. 删除无关检查后，确认该测试返回 `PASS` 且无复核项。
3. 运行前置检查、命令行集成测试和完整测试集。
4. 搜索已删除类别与辅助函数，确认生产代码、测试和文档中均无残留。
5. 检查最终差异和提交内容，仅包含本设计直接要求的修改。
