# Episode 采样与 README 重构设计

## 目标

RoboMetaNorm 在规范化时最多读取两个代表性 episode 的 Parquet 和视频内容，降低大规模数据集的分析开销；同时重构 README，使项目定位、使用方式、处理边界和输出结果更容易理解。

## 采样规则

- 所有 episode 文件按路径稳定排序。
- 没有文件时沿用现有的缺失数据处理。
- 只有一个文件时使用该文件。
- 两个及以上文件时使用首个和末个文件。
- 为确定首尾文件，程序仍会遍历目录项；未被选中的文件不会进入 Parquet 画像、FFprobe、视频抽帧或 VLM 判断。

采样规则由一个无状态的通用函数实现，Parquet 和视频链路共用，避免两处策略漂移。采样数量固定为两个，不新增命令行配置。

## Parquet 数据流

数据集管线发现并排序 Parquet 路径后立即采样，只把选中的路径交给画像缓存。缓存指纹只包含选中的文件及采样配置，因此未参与分析的中间 episode 发生变化时不会触发无意义的缓存重建。

每个入选 Parquet 继续沿用现有读取上限：读取 schema、文件元数据和首个最多 512 行的 batch。跨 episode 布局一致性仅代表这两个样本 episode，进度和缓存命中数量也显示实际采样数。

## 视频数据流

相机媒体发现完成后立即应用同一首尾采样规则。媒体校验、FFprobe、两阶段抽帧及 VLM 只能接收到这两个路径。第一阶段证据充分时可以只读取首个 episode；需要第二阶段时最多使用首尾两个 episode，不再选择中间 episode。

现有媒体缺失、不可读、FPS/shape 不一致和低置信度复核行为保持不变。

## README 结构

README 按用户任务组织为以下部分：

1. 项目定位与安全保证。
2. 支持范围与明确不处理的内容。
3. 环境要求和安装方式。
4. 输入目录约定。
5. 扫描、规范化和可选 VLM 的快速命令。
6. Episode 采样及判定规则。
7. 输出文件、状态和人工复核说明。
8. 测试命令。

主流程使用通用占位符，不绑定具体数据集或供应商；供应商特有行为仅保留为简短兼容性说明。README 不使用内部里程碑编号解释功能。

## 验证

- 单元测试覆盖空序列、单文件、双文件和多文件的确定性首尾选择。
- Parquet 管线测试证明三个及以上 episode 时画像函数只接收两个路径。
- 视频测试证明媒体发现结果和 VLM 第二阶段最多包含首尾两个 episode。
- 缓存测试证明指纹只受入选 Parquet 影响。
- 命令行集成测试验证多 episode 数据集只报告两个画像进度。
- 运行完整测试集确认现有命名、复核和输出行为不回归。
