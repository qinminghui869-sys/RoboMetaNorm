# README 输入与输出章节整理设计

## 目标

重组 README 的“输入与输出”内容，使其按数据流展开，具备明确的信息层级、严谨的技术语义和简洁的中文表达，同时保持现有程序行为与输出契约不变。

## 信息结构

### 输入与输出

该章节仅回答四个问题：

1. 数据集必须提供哪些目录和文件；
2. 程序读取哪些有限证据；
3. `normalize` 生成哪些文件，各文件承担什么职责；
4. 哪些异常会使输出不完整，以及哪些源数据始终不会被改写。

内容按“输入要求—证据范围—输出文件—写出边界”的顺序组织。输入要求区分必需项与可选项；输出文件使用紧凑列表，统一描述其用途、确认状态和复核信息。

### 输出语义与校验规则

从原“输入与输出”章节迁出以下实现语义：

- `robo_annotation.yaml` 的顶层结构；
- `PASS` 与 `REVIEW` 的判定及 `review` 字段；
- REVIEW 状态下的本地尽力推导通道；
- 相机与机器通道命名；
- 关节布局预检及阻断条件。

该章节按“复核状态—通道语义—预检规则”组织。首次出现 best-effort 时采用中文“尽力推导”解释，英文仅作为括注，避免中英文术语混杂。

## 表达原则

- 先陈述一般规则，再说明异常情况；
- 每个段落只表达一个中心含义；
- 删除与“工作流程”重复的身份来源和 VLM 调用说明，仅保留必要引用；
- 使用可验证的限定语，不将已处理业务失败扩大为所有异常；
- 保留文件名、字段名、状态名和通道名等必要技术标识；
- 不引入新的功能、参数、输出文件或行为承诺。

## 验收标准

- “输入与输出”可独立回答输入、输出及写出边界；
- 详细规则集中在“输出语义与校验规则”，不与产物列表混杂；
- `robot_type`、单次 VLM、PASS/REVIEW、YAML fallback 和异常边界与当前实现一致；
- Markdown 层级、列表和标点一致；
- README 其他章节不发生无关改动。
